# Implementation Plan: Academic Paper Search Integration

**Branch**: `001-academic-paper-search` | **Date**: 2026-03-07 | **Spec**: [spec.md](./spec.md)
**Input**: Feature specification from `/specs/001-academic-paper-search/spec.md`

## Summary

Add OpenAlex and arXiv APIs as academic paper search sources to the research pipeline. When the query router classifies a query as `academic`, `technical`, or `explanatory`, dispatch academic paper searches in parallel with Tavily web searches. Academic results are converted to the standard `TavilySearchResult` format and flow through the existing extraction, synthesis, gap analysis, and caching pipeline unchanged. The Source type is extended with optional academic metadata fields for UI display.

## Technical Context

**Language/Version**: TypeScript 5.x (Next.js 15.2 App Router)
**Primary Dependencies**: Next.js 15.2, React 19, existing `api-utils.ts` utilities
**Storage**: Existing Supabase cache (no schema changes)
**Testing**: Jest (existing test infrastructure)
**Target Platform**: Vercel serverless (Node.js runtime)
**Project Type**: Web application (Next.js full-stack)
**Performance Goals**: Academic searches must not add >500ms to pipeline latency (parallel execution)
**Constraints**: OpenAlex 10 req/s rate limit; arXiv 1 req/3s rate limit; serverless function timeouts
**Scale/Scope**: 3-4 OpenAlex calls + 1 arXiv call per research query; free APIs, no credit impact
